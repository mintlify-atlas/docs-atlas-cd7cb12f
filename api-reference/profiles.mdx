---
title: 'HTTPower.Profiles'
description: 'Pre-built configuration profiles for common use cases.'
---

`HTTPower.Profiles` provides named, opinionated configurations that encode best practices for middleware coordination. Each profile is a keyword list you can pass directly to `HTTPower.new/1` or merge with your own overrides.

## Functions

### `list/0`

Returns the names of all available profiles.

```elixir
@spec list() :: [atom()]
```

```elixir
HTTPower.Profiles.list()
# => [:payment_processing, :high_volume_api, :microservices_mesh]
```

---

### `get/1`

Fetches a profile configuration by atom name.

```elixir
@spec get(atom()) :: {:ok, keyword()} | {:error, :unknown_profile}
```

<ParamField path="name" type="atom" required>
  Profile name. Must be one of `:payment_processing`, `:high_volume_api`, or `:microservices_mesh`.
</ParamField>

**Returns** `{:ok, keyword()}` for valid profile names, or `{:error, :unknown_profile}` for unrecognized names.

```elixir
{:ok, config} = HTTPower.Profiles.get(:payment_processing)
client = HTTPower.new([base_url: "https://pay.example.com"] ++ config)

{:error, :unknown_profile} = HTTPower.Profiles.get(:nonexistent)
```

---

### `payment_processing/0`

Returns the configuration for payment gateways and financial APIs. Prioritizes correctness and reliability over raw throughput.

```elixir
@spec payment_processing() :: keyword()
```

**Best for:** Stripe, PayPal, payment gateways, financial APIs.

```elixir
HTTPower.Profiles.payment_processing()
# Returns:
[
  rate_limit: [
    enabled: true,
    requests: 100,
    per: :minute,
    strategy: :wait,
    max_wait_time: 5000,
    adaptive: true
  ],
  circuit_breaker: [
    enabled: true,
    failure_threshold: 6,
    failure_threshold_percentage: 30.0,
    window_size: 20,
    timeout: 30_000,
    half_open_requests: 3
  ],
  deduplicate: [
    enabled: true,
    ttl: 5_000
  ],
  max_retries: 3,
  base_delay: 2_000,
  max_delay: 30_000
]
```

<Note>
Deduplication with a 5-second TTL is enabled by default to prevent double charges from retry storms or double-click events.
</Note>

### `high_volume_api/0`

Returns the configuration for high-throughput public or internal APIs. Prioritizes maximum request volume over individual latency.

```elixir
@spec high_volume_api() :: keyword()
```

**Best for:** Public APIs, high-traffic internal services, content APIs.

```elixir

    adaptive: true
  ],
  circuit_breaker: [
    enabled: true,
    failure_threshold: 50,
    failure_threshold_percentage: 50.0,
    window_size: 100,
    timeout: 5_000,
    half_open_requests: 10
  ],
  deduplicate: [
    enabled: true,
    ttl: 1_000
  ],
  max_retries: 2,
  base_delay: 500,
  max_delay: 10_000
```

### `microservices_mesh/0`

Returns the configuration for service-to-service calls in a microservices architecture. Prioritizes resilience and preventing cascade failures.

```elixir
@spec microservices_mesh() :: keyword()
```

**Best for:** Service-to-service calls, internal microservices, Kubernetes environments.

```elixir

# Returns:

  rate_limit: [
```

## Usage

<CodeGroup>

```elixir apply a profile directly
client = HTTPower.new(
  base_url: "https://api.stripe.com",
  profile: :payment_processing
)
```

```elixir override individual values
# Profile settings are merged with explicit options; explicit options win.
client = HTTPower.new(
  base_url: "https://api.example.com",
  profile: :high_volume_api,
  rate_limit: [requests: 2000]  # overrides the profile's 1000 req/min
)
```

```elixir use get/1 for programmatic access
{:ok, config} = HTTPower.Profiles.get(:microservices_mesh)

# Inspect or modify before passing to the client
client = HTTPower.new([base_url: "https://svc.internal"] ++ config)
```

</CodeGroup>
