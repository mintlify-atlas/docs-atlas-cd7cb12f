<Steps>

<Step title="Add dependencies">

Add `httpower` and at least one HTTP client adapter to `mix.exs`. Finch is recommended for production use.

<Tabs>
  <Tab title="Finch (recommended)">
    ```elixir mix.exs
    def deps do
      [
        {:httpower, "~> 0.20.0"},
        {:finch, ">= 0.19.0"}
      ]
    end
    ```
  </Tab>
  <Tab title="Req">
    ```elixir mix.exs
    def deps do
      [
        {:httpower, "~> 0.20.0"},
        {:req, ">= 0.4.0"}
      ]
    end
    ```
  </Tab>
  <Tab title="Tesla">
    ```elixir mix.exs
    def deps do
      [
        {:httpower, "~> 0.20.0"},
        {:tesla, ">= 1.10.0"}
      ]
    end
    ```
  </Tab>
</Tabs>

Then fetch dependencies:

```bash
mix deps.get
```

<Note>
HTTPower requires at least one adapter. If multiple adapters are present, Finch is used by default. You can override this with the `adapter` option.
</Note>

</Step>

<Step title="Start the Finch supervisor (Finch adapter only)">

If you are using the Finch adapter, add `HTTPower.Finch` to your application's supervision tree.

```elixir lib/my_app/application.ex
defmodule MyApp.Application do
  use Application

  def start(_type, _args) do
    children = [
      {Finch, name: HTTPower.Finch}
    ]

    opts = [strategy: :one_for_one, name: MyApp.Supervisor]
    Supervisor.start_link(children, opts)
  end
end
```

<Note>
This step is only required when using the Finch adapter. Skip it if you are using Req or Tesla.
</Note>

</Step>

<Step title="Make your first request">

HTTPower always returns a result tuple — it never raises on HTTP errors.

```elixir
{:ok, response} = HTTPower.get("https://api.example.com/users")

IO.inspect(response.status)  # 200
IO.inspect(response.body)    # %{"users" => [...]}
```

`{:ok, response}` means the server replied, not necessarily that the request succeeded in a business logic sense. A 404 or 500 response is still returned as `{:ok, response}`. Always check `response.status`.

</Step>

<Step title="Create a reusable client">

Use `HTTPower.new/1` to create a pre-configured client you can reuse across multiple requests. This is especially useful for API integrations that share a base URL, authentication headers, or common options.

```elixir
client = HTTPower.new(
  base_url: "https://api.example.com",
  headers: %{"authorization" => "Bearer #{token}"},
  timeout: 30,
  max_retries: 3
)

{:ok, users}   = HTTPower.get(client, "/users")
{:ok, user}    = HTTPower.get(client, "/users/123")
{:ok, created} = HTTPower.post(client, "/users", body: data)
```

</Step>

<Step title="Handle errors">

HTTPower never raises for HTTP operations. Match on the result tuple to handle all outcomes, including network errors and 4xx/5xx responses.

```elixir
case HTTPower.get("https://api.example.com/users") do
  {:ok, %HTTPower.Response{status: 200, body: body}} ->
    # Success — process response body
    process_data(body)

  {:ok, %HTTPower.Response{status: 404}} ->
    # Server responded, resource not found
    handle_not_found()

  {:ok, %HTTPower.Response{status: status}} when status in 500..599 ->
    # Server responded with an error (after retries exhausted)
    handle_server_error(status)

  {:error, %HTTPower.Error{reason: :timeout}} ->
    # Network timeout
    handle_timeout()

  {:error, %HTTPower.Error{reason: :econnrefused}} ->
    # Connection refused
    handle_connection_error()
end
```

<Tip>
Configuration errors — such as passing an unknown `:profile` to `HTTPower.new/1` — raise `ArgumentError` at client construction time, following standard Elixir conventions.
</Tip>

</Step>

<Step title="Set up test mode">

HTTPower can block all real HTTP requests during testing. Enable test mode in `test/test_helper.exs` and use `HTTPower.Test` to stub responses.

```elixir test/test_helper.exs
Application.put_env(:httpower, :test_mode, true)
ExUnit.start()
```

```elixir test/my_app_test.exs
defmodule MyAppTest do
  use ExUnit.Case

  setup do
    HTTPower.Test.setup()
  end

  test "API integration with mocking" do
    HTTPower.Test.stub(fn conn ->
      HTTPower.Test.json(conn, %{status: "success"})
    end)

    {:ok, response} = HTTPower.get("https://api.example.com/test")
    assert response.body == %{"status" => "success"}
  end

  test "real requests are blocked" do
    {:error, error} = HTTPower.get("https://real-api.com")
    assert error.reason == :network_blocked
  end
end
```

`HTTPower.Test` is adapter-agnostic — your tests stay valid even if you switch adapters later.

</Step>
